Clean Arquitecture

Imagine un diseño arquitectonico capaz de entregar sencillez y mantenibilidad a traves del tiempo.
Este enfoque busca crear sistemas de software bien estructurados, mantenibles y escalables al organizar el código en capas y establecer reglas claras de dependencias entre ellas.


Reglas:

  • Existen capas:
    * Capa de dominio: "Casos de uso, persistencia, Entidades"

  • Separa el dominio(Objetos, Métodos, Clases, Casos de uso) de la infrastructura (librerias, database)

  • A traves de interfaces se completa el funcionamiento del software (Inyeción de dependencias, Segregación de interfaces)

  •           # Estructura de las carpetas en java sprintboot project

          #ProjectName 
          |---application
          |       |---src
          |       |----|-main //RUN
          | 
          |---domain
          |-----|--entities/src/ //house, person, obj 
          |-----|--database/src/ //interface conector db
          |-----|--usecases/src/ //save user, delete client, ...
          |
          |
          |---infracstructure
          |--------|-----|drivenAdapters/src  // Specify SQL, Objeto de Acceso a Datos, 
          |--------|-----|entryPoints/src //all API urls
